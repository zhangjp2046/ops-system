## 1. Linux服务器

### 方案一：SSH采集（推荐）

#### 服务器端配置

**1. 开启SSH服务（通常已默认开启）**
```bash
# 检查SSH服务状态
systemctl status sshd

# 如果未开启
systemctl start sshd
systemctl enable sshd
```

**2. 创建监控专用用户（可选，更安全）**
```bash
# 创建只读用户
useradd -m -s /bin/bash monitor
passwd monitor

# 限制该用户只能执行特定命令
# 编辑 /etc/ssh/sshd_config
AllowUsers monitor

# 重启SSH
systemctl restart sshd
```

**3. 在服务器放置采集脚本**
```bash
# 创建采集脚本目录
sudo mkdir -p /opt/monitor/scripts

# 创建采集脚本
sudo cat > /opt/monitor/scripts/status.sh << 'EOF'
#!/bin/bash
# 输出服务器状态JSON - 用于SSH采集

# 获取CPU使用率（2秒平均值）
CPU=$(top -bn2 -d 0.5 | grep "Cpu(s)" | tail -1 | awk '{print $2}')
# 去掉us后缀
CPU=${CPU%us*}}

# 获取内存信息
MEM_TOTAL=$(free -m | grep Mem | awk '{print $2}')
MEM_USED=$(free -m | grep Mem | awk '{print $3}')
MEM_FREE=$(free -m | grep Mem | awk '{print $4}')
MEM_CALC=$(awk "BEGIN {printf \"%.1f\", ($MEM_USED/$MEM_TOTAL)*100}")

# 获取磁盘使用率
DISK=$(df -h / | tail -1 | awk '{print $5}' | sed 's/%//')

# 获取负载
LOAD=$(uptime | awk -F'load average:' '{print $2}' | tr -d ' ')

# 获取运行时间
UPTIME=$(uptime -p 2>/dev/null || uptime)

# 获取在线用户
USERS=$(who | wc -l)

# 输出JSON
cat << JSON
{
    "hostname": "$(hostname)",
    "cpu_usage": "$CPU",
    "mem_total": "$MEM_TOTAL",
    "mem_used": "$MEM_USED", 
    "mem_free": "$MEM_FREE",
    "mem_usage": "$MEM_CALC",
    "disk_usage": "$DISK",
    "load_avg": "$LOAD",
    "uptime": "$UPTIME",
    "online_users": $USERS,
    "timestamp": "$(date -u '+%Y-%m-%dT%H:%M:%SZ')"
}
JSON
EOF

# 设置权限
sudo chmod +x /opt/monitor/scripts/status.sh
sudo chown monitor:monitor /opt/monitor/scripts/status.sh
```

#### 平台端测试

```bash
# 使用SSH协议测试
curl -X POST http://localhost:8000/api/monitoring/quick-test/ \
  -H "Content-Type: application/json" \
  -d '{
    "protocol": "ssh",
    "host": "192.168.1.199",
    "port": 22,
    "username": "monitor",
    "password": "your_password"
  }'
```

#### 平台端配置保存

```bash

  -H "Content-Type: application/json" \
```

### 方案二：SNMP采集

#### 服务器端配置

```bash
# Ubuntu/Debian
sudo apt update
sudo apt install snmpd snmp

# 编辑SNMP配置
sudo vi /etc/snmp/snmpd.conf

# 找到以下行，修改为：
# agentAddress  udp:127.0.0.1:161
# 改为监听所有地址：
agentAddress  udp:161,udp6:[::1]:161

# 找到 com2sec 行，修改community（默认public不安全）：
# 格式：com2sec <community name> <source> <community string>
com2sec public  default    your_community_name

# 找到 access 行，确保有只读权限：
access public  ""   any       noauth    exact  all  none  none

# 找到 view 行，确保包含系统信息：
view all    included  .1        80

# 重启SNMP服务
sudo systemctl restart snmpd

# 测试本地SNMP
snmpwalk -v 2c -c your_community_name localhost .1
```

#### Windows防火放行
```bash
# 如果有防火墙，放行UDP 161端口
sudo ufw allow 161/udp
```

#### 平台端测试
```bash
curl -X POST http://localhost:8000/api/monitoring/quick-test/ \
  -H "Content-Type: application/json" \
  -d '{
    "protocol": "snmp",
    "host": "192.168.1.199",
    "port": 161,
    "community": "your_community_name"
  }'
```

### 方案二：PowerShell over SSH/WinRM

#### 服务器端配置

**1. 开启WinRM服务**

```powershell
# 以管理员身份运行PowerShell
Enable-PSRemoting -Force

# 配置为信任平台服务器
Set-Item WSMan:\localhost\Client\TrustedHosts -Value "你的平台服务器IP" -Force

# 重启服务
Restart-Service WinRM
```

**2. 创建监控脚本**

```powershell
# 创建目录
New-Item -ItemType Directory -Path C:\Monitor -Force

# 创建采集脚本 C:\Monitor\status.ps1
$script = @'
# 输出服务器状态JSON
$cpu = (Get-Counter '\Processor(_Total)\% Processor Time').CounterSamples.CookedValue
$cpu = [math]::Round($cpu, 1)

$os = Get-CimInstance Win32_OperatingSystem
$memTotal = [math]::Round($os.TotalVisibleMemorySize / 1MB, 2)
$memFree = [math]::Round($os.FreePhysicalMemory / 1MB, 2)
$memUsed = $memTotal - $memFree
$memUsage = [math]::Round(($memUsed / $memTotal) * 100, 1)

$disk = Get-CimInstance Win32_LogicalDisk -Filter "DeviceID='C:'"
$diskUsage = [math]::Round(($disk.Size - $disk.FreeSpace) / $disk.Size * 100, 1)

$uptime = (Get-Date) - $os.LastBootUpTime
$uptimeStr = "$($uptime.Days)天 $($uptime.Hours)小时"

$result = @{
    hostname = $env:COMPUTERNAME
    cpu_usage = $cpu
    mem_total = $memTotal
    mem_used = $memUsed
    mem_free = $memFree
    mem_usage = $memUsage
    disk_usage = $diskUsage
    uptime = $uptimeStr
    timestamp = (Get-Date -Format "yyyy-MM-ddTHH:mm:ssZ")
}

$result | ConvertTo-Json
'@

$script | Out-File -FilePath C:\Monitor\status.ps1 -Encoding UTF8
```


## 3. 网络设备

### 支持的设备类型

- 交换机（华为、H3C、Cisco、锐捷）
- 路由器
- 无线AC/AP

### 通用配置

#### SNMP配置（所有网络设备通用）

```bash
# Cisco设备
snmp-server community your_community_name RO
snmp-server host 你的平台IP version 2c your_community_name
snmp-server location Hospital_Datacenter
snmp-server contact admin@hospital.com

# 华为设备
snmp-agent community-read your_community_name
snmp-agent target-host host-name monitor address 你的平台IP udp-port 162
snmp-agent sys-info location Hospital_Datacenter

# H3C设备
snmp-agent community-read your_community_name
snmp-agent target-host host-name monitor address 你的平台IP
```

#### SSH配置（CLI采集）

```bash
# Cisco设备
ip domain-name hospital.com
crypto key generate rsa modulus 2048
line vty 0 4
 login local
 transport input ssh
```

### 平台端测试

```bash
# 测试网络设备SNMP
curl -X POST http://localhost:8000/api/monitoring/quick-test/ \
  -H "Content-Type: application/json" \
  -d '{
    "protocol": "snmp",
```

## 4. 安全设备

### 防火墙

#### Hillstone（山石网科）

```bash
# 开启SNMP
snmp-server enable
snmp-server community hospital_read
snmp-server host 你的平台IP

# 或通过Web管理界面：
# 系统 → SNMP → 启用SNMPv2c
```

#### 深信服（Sangfor）

```bash
# 开启SNMP
snmp-agent community-read hospital_snmp
snmp-agent target-host 你的平台IP

# 通过Web界面：
# 策略 → SNMP → 添加社区名
```

#### Fortinet（飞塔）

```bash
# CLI配置
config system snmp community
    edit 1
        set name "hospital_monitor"
        config hosts
            edit 1
                set ip 你的平台IP/32
            next
        end
    next
end
```

#### Palo Alto

```bash
# 开启SNMP
# Devices → Management Interface Settings → SNMP
# 添加Community String

# 或通过API
curl -X POST "https://firewall/api/" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d '{"type": "config", "action": "set", ...}'
```

### IDS/IPS（入侵检测/防御系统）

#### 绿盟（NSFOCUS）

```bash
# 开启SNMP
snmpd enable
snmpd community hospital_read
snmpd host 你的平台IP
```

#### Snort/Suricata（开源）

```bash
# 配置SNMP
# 编辑 /etc/snmp/snmpd.conf
rocommunity hospital_read
snmpd enable
```

### 堡垒机

#### 常用配置

```bash
# 开启SNMP
snmp-server community hospital_bastion RO
snmp-server location Hospital_Bastion_Host
```

## 故障排查

### Ping不通
```bash
# 检查网络连通性
ping 192.168.1.199

# 检查防火墙
sudo ufw status
sudo iptables -L -n
```

### SNMP不通
```bash
# 检查SNMP服务
systemctl status snmpd

# 检查端口
sudo netstat -ulnp | grep 161

# 测试SNMPWalk
snmpwalk -v 2c -c public 192.168.1.199 .1
```

### SSH连接失败
```bash
# 检查SSH服务
systemctl status sshd

# 检查端口
telnet 192.168.1.199 22

# 检查用户权限
grep monitor /etc/passwd
```

### 数据库连接失败
```bash
# 检查端口
telnet 192.168.1.11 3306

# 检查用户权限
mysql -u root -p -e "SHOW GRANTS FOR 'monitor'@'你的IP';"
```
