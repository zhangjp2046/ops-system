# 资产巡检内容规划

## 一、资产类型与巡检项对照表

### 1. 服务器 (SERVER)
```
检查项:
  1. 网络连通性 (ping)
  2. SSH端口 (22)
  3. Web端口 (80/443)
  4. 系统运行时间
  5. CPU使用率 (>80% 警告, >95% 危险)
  6. 内存使用率 (>80% 警告, >95% 危险)
  7. 磁盘使用率 (>80% 警告, >95% 危险)
  8. 系统日志错误 (最近1小时有ERROR级别日志)
  9. 服务运行状态 (关键服务是否Running)
```

### 2. 网络设备 (NETWORK)
```
检查项:
  1. 网络连通性 (ping)
  2. SNMP端口 (161)
  3. 设备可管理性 (snmpwalk)
  4. CPU使用率 (>70% 警告, >85% 危险)
  5. 内存使用率 (>70% 警告, >85% 危险)
  6. 端口状态 (up/down)
  7. 带宽利用率 (>70% 警告, >90% 危险)
  8. 错误包率 (>1% 警告, >5% 危险)
  9. 设备温度 (如支持)
```

### 3. 数据库 (DATABASE) - MySQL / MSSQL / Oracle / PostgreSQL
```
MySQL:
  1. 数据库连接
  2. 数据库版本
  3. 数据库大小
  4. 连接数使用率 (>80% 警告)
  5. 缓冲池命中率 (<90% 警告)
  6. 慢查询数量 (>10条/小时 警告)
  7. 备份状态
  8. 复制状态 (主从)
  9. 错误日志检查

MSSQL:
  1. 数据库连接
  2. 数据库版本
  3. 数据库列表
  4. 数据库文件大小
  5. 会话数
  6. 缓冲命中率
  7. 备份状态
  8. 死锁/锁信息
  9. 慢查询

Oracle:
  1. 数据库连接
  2. 数据库版本
  3. 表空间使用率 (>80% 警告)
  4. 数据文件大小
  5. 会话数
  6. 缓冲命中率
  7. 归档日志状态
  8. RMAN备份状态
  9. ORA-错误日志
  10. 资源消耗SQL
  11. 锁信息

PostgreSQL:
  1. 数据库连接
  2. 数据库版本
  3. 数据库大小
  4. 连接数
  5. 缓冲命中率
  6. 慢查询
  7. 复制状态
```

### 4. 中间件 (MIDDLEWARE)
```
Nginx:
  1. 网络连通性 (ping)
  2. HTTP端口 (80/443)
  3. 服务状态 (nginx活着?)
  4. 连接数使用率
  5. 请求成功率 (5xx比例 >1% 警告)
  6. upstream响应时间

Redis:
  1. 网络连通性
  2. 端口 (6379)
  3. 连接状态 (ping)
  4. 内存使用率 (>80% 警告)
  5. 持久化状态 (RDB/AOF)
  6. 复制状态
  7. 过期key扫描

Kafka:
  1. 网络连通性
  2. Zookeeper连接
  3. Broker健康状态
  4. Topic数量
  5. 消费Lag (>10000 警告)
  6. ISR副本状态

RabbitMQ:
  1. 网络连通性
  2. 管理端口 (15672)
  3. 队列消息数 (>10000 警告)
  4. 内存使用率
  5. 磁盘使用率
  6. Erlang进程数
```

### 5. API接口 (API)
```
检查项:
  1. API可访问性 (HTTP ping)
  2. 响应时间 (>3s 警告, >10s 危险)
  3. HTTP状态码 (非2xx 警告)
  4. SSL证书有效期 (<30天 警告)
  5. 接口认证有效性
  6. 返回数据格式正确性
```

### 6. 云服务 (CLOUD_SERVICE)
```
检查项:
  1. API端点可访问性
  2. 认证有效性 (Token/Key有效)
  3. 资源配额使用率
  4. 账单/计费异常
  5. 服务健康状态 (AWS/Azure/GCP Status Page)
```

### 7. 存储设备 (STORAGE)
```
检查项:
  1. 网络连通性 (ping)
  2. 管理端口
  3. 存储池使用率 (>80% 警告)
  4. 磁盘/SSD寿命
  5. RAID状态 (降级 警告, 故障 危险)
  6. 复制/快照状态
  7. 控制器健康状态
```

### 8. 应用程序 (APPLICATION)
```
检查项:
  1. 服务可访问性 (HTTP/HTTPS)
  2. 进程存活状态
  3. 响应时间 (>5s 警告)
  4. 错误率 (>1% 警告)
  5. 数据库连接池使用率
  6. JVM堆内存使用率 (Java)
  7. 日志错误检查 (最近1小时ERROR)
```

---

## 二、巡检状态判断标准

| 级别 | Tag颜色 | 条件 |
|------|---------|------|
| pass | 绿色 | 检查通过 |
| warning | 黄色/橙色 | 超过阈值警告 |
| fail | 红色 | 超过阈值危险或检查失败 |
| skip | 灰色 | 跳过(不支持该项目) |
