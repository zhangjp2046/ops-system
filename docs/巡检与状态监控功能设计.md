# 巡检与状态监控功能设计

## 一、功能概述

本系统提供设备状态监控和巡检功能，支持：
- 设备在线/离线状态自动刷新
- SNMP协议设备巡检
- 数据库巡检（MySQL、MSSQL、Oracle）
- 手工和定时巡检触发
- 巡检结果记录和查询

---

## 二、定时任务

### 2.1 定时任务列表

| 任务名称 | 类型 | 目标 | 间隔 | 说明 |
|---------|------|------|------|------|
| 全局Ping监控 | monitoring | 全部 | 5分钟 | ICMP ping检测 |
| 全局端口检测 | monitoring | 全部 | 10分钟 | TCP端口检测 |
| 设备状态刷新 | status_refresh | 全部 | 5分钟 | 在线/离线状态 |
| 巡检-藏书卫生院-SNMP设备 | inspection | 客户#1 | 2小时 | SNMP设备巡检 |
| 巡检-太湖度假区医院-SNMP设备 | inspection | 客户#2 | 2小时 | SNMP设备巡检 |
| 数据库日常巡检 | inspection | 全部 | 4小时 | 数据库状态巡检 |
| 每日资产统计报表 | report | 全部 | 1天 | 资产统计报表 |
| 每月数据清理 | cleanup | 全部 | 1天 | 清理过期数据 |

### 2.2 状态刷新任务 (status_refresh)

**执行内容：**
- Ping检测设备连通性
- 更新资产在线状态（online字段）
- 记录状态变更历史
- 更新最后检查时间（last_check_time）

**结果字段：**
```json
{
  "total": 5,      // 总设备数
  "online": 2,     // 在线数
  "offline": 3     // 离线数
}
```

## 三、巡检类型

### 3.1 SNMP设备巡检

**触发方式：**
- 手工触发：POST /api/inspection/run/
- 定时任务：按配置的间隔自动执行
- 批量触发：POST /api/inspection/batch-run/

**巡检项目：**

| 项目代码 | 项目名称 | 类别 | 说明 |
|---------|---------|------|------|
| SNMP_CONNECTION | SNMP连接 | connection | SNMP服务可用性 |
| SYSTEM_INFO | 系统信息 | info | 设备型号、系统描述 |
| UPTIME | 运行时间 | info | 设备运行时间 |
| INTERFACE_STATUS | 接口状态 | network | 网络接口UP/DOWN |
| INTERFACE_COUNT | 接口数量 | network | 端口总数 |
| BANDWIDTH_USAGE | 带宽使用 | performance | 带宽利用率 |

**巡检结果状态：**
- PASS: 通过
- WARNING: 警告（需要关注）
- FAIL: 失败（需要处理）

### 3.2 数据库巡检

**支持数据库：**
- MySQL
- Microsoft SQL Server (MSSQL)
- Oracle

**巡检项目：**

| 项目代码 | 项目名称 | 类别 | 说明 |
|---------|---------|------|------|
| DB_CONNECTION | 数据库连接 | connection | 连接可用性 |
| DB_VERSION | 数据库版本 | info | 版本信息 |
| TABLESPACE | 表空间使用率 | storage | 存储空间 |
| BUFFER_HIT | 缓冲池命中率 | performance | 缓存效率 |
| SESSIONS | 活跃会话数 | session | 并发连接 |
| BACKUP | 备份状态 | backup | 备份执行情况 |
| ERROR_LOG | 错误日志 | log | 错误记录数 |

**连接配置（从资产字段获取）：**
```
db_host / host_address  -> 主机地址
db_port / port          -> 端口
db_name / database_name  -> 数据库名
db_username / username  -> 用户名
db_password / password  -> 密码
```

## 四、API接口

### 4.1 手工执行巡检

**接口：** `POST /api/inspection/run/`

**参数：**
```json
{
  "asset_id": 1,           // 资产ID (必填)
  "inspection_type": "SNMP"  // 巡检类型: SNMP / DATABASE (默认SNMP)
}
```

**响应：**
```json
{
  "success": true,
  "message": "巡检执行成功",
  "data": {
    "inspection_id": 8,
    "status": "COMPLETED",
    "passed_items": 5,
    "warning_items": 1,
    "failed_items": 1,
    "summary": "发现1项异常，需要关注"
  }
}
```

### 4.2 批量执行巡检

**接口：** `POST /api/inspection/batch-run/`

**参数：**
```json
{
  "customer_id": 1,           // 客户ID (0表示全部)
  "asset_ids": [1, 2, 3],     // 资产ID列表 (可选)
  "inspection_type": "SNMP"     // 巡检类型
}
```

### 4.3 获取巡检详情

**接口：** `GET /api/inspection/status/<inspection_id>/`

**响应：**

### 4.4 巡检记录列表

**接口：** `GET /api/inspection/list/`

**参数：**
- customer_id: 客户ID (可选)
- asset_id: 资产ID (可选)
- type: 巡检类型 (可选)
- status: 状态 (可选)
- page: 页码 (默认1)
- page_size: 每页数量 (默认20)

## 六、巡检执行流程

### 6.1 SNMP巡检流程

```
1. 创建Inspection记录 (status=RUNNING)
2. 获取资产SNMP配置 (community, port)
3. 执行SNMP GET请求获取系统信息
4. 执行SNMP WALK请求获取接口信息
5. 分析结果，与阈值比较
6. 生成巡检项目记录
7. 更新Inspection状态
```

### 6.2 数据库巡检流程

```
1. 创建Inspection记录 (status=RUNNING)
2. 根据资产类型选择连接器 (MySQL/MSSQL/Oracle)
3. 获取连接配置 (host, port, database, username, password)
4. 建立数据库连接
5. 执行巡检SQL查询
6. 分析结果，与阈值比较
7. 生成巡检项目记录
8. 关闭数据库连接
9. 更新Inspection状态
```

## 七、命令行工具

### 7.1 巡检命令

```bash
# 执行SNMP设备巡检
python manage.py run_inspection snmp --customer-id=1

# 执行数据库巡检
python manage.py run_inspection database --customer-id=1

# 刷新设备状态
python manage.py run_inspection status

# 执行所有巡检
python manage.py run_inspection all --customer-id=1
```
