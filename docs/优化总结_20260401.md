## 二、页面功能修复

### 1. Dashboard (首页)
**问题**: 离线资产不更新、最近巡检无明细、任务执行不刷新

**修复**:
- 刷新按钮：先 ping 所有资产（更新 online 状态）再加载统计
- 每 30 秒自动刷新所有数据（含任务执行记录）
- 最近巡检表格字段修正：`pass_checks / fail_checks / warning_checks`
- 告警严重程度显示修复（支持小写 `warning`）

### 2. 资产清单 (资产管理)
**问题**: 在线状态和资产状态显示不明确

**修复**:
- 新增"在线状态"列（绿色 = 在线 / 红色 = 离线），显示 ping 检测结果
- 刷新按钮：ping 检测所有资产 → 更新在线状态 → 刷新列表
- 每 60 秒自动刷新（含 ping 检测）
- `AssetSerializer` 添加 `online` 和 `last_check_time` 字段

### 3. 资产表单 (新建/编辑资产)
**问题**: 无法选择客户

**修复**:
- 新建资产时默认选中第一个客户
- axios 替换为带拦截器的 api 实例

### 4. 巡检计划 (InspectionPlanList)
**问题**: 无法获取客户信息、不能检索创建的巡检计划

**修复**:
- 所有 axios 调用改为 api（含拦截器的实例，直接返回 data）
- 新建计划时默认选中第一个客户
- 列表页添加客户筛选下拉框和客户列
- `InspectionPlan` 模型添加 `customer` 外键
- `get_queryset` 支持 `customer` 参数过滤

### 5. 采集测试 (MonitorTest)
**问题**: 结果显示不明确

**修复**:
- 错误信息多行格式化（替换 `\n` 为真正换行）
- 数据库指标增加"驱动方式"、"响应时间"字段
- 协议标签颜色优化（mysql/mssql 去掉 danger）
- MSSQL 优先使用 tsql（与巡检任务一致的 FreeTDS 方案），pyodbc/pymssql 降为备用

### 6. 路由/菜单
**问题**: `/monitoring`、`/monitoring/alerts`、`/scheduler`、`/inspection` 页面打不开

**修复**:
- 添加缺失路由和占位页面
- 侧边栏菜单重构：监控中心 → 告警中心 → 巡检管理(子菜单) → 定时任务
- 添加 `Monitor`、`Bell`、`Timer` 等图标

## 三、巡检架构完善

### execute 方法重构
根据资产类型自动选择巡检方式：

| 资产类型 | 巡检内容 |
|----------|----------|
| `DATABASE` | 数据库完整巡检（连接/版本/会话/缓冲池/备份/锁信息等） |
| 其他资产 | ping + 端口检测 |

### 巡检内容规划文档
`docs/巡检内容规划.md` — 规划了 8 类资产（SERVER/NETWORK/DATABASE/MIDDLEWARE/API/CLOUD_SERVICE/STORAGE/APPLICATION）的检查项和状态判断标准
